## Regras de edição (anti-crash)

- Commitar a cada ~5 arquivos editados com mensagem descritiva. NÃO acumular 10+ edições sem commit
- Após cada commit intermediário, rodar `npm run build` pra validar antes de continuar
- Se o build falhar, corrigir ANTES de editar mais arquivos
- Evitar refactors que toquem em mais de 8 arquivos de uma vez — dividir em etapas

## Deploy

- Sempre push após alterações para atualizar o site no Render (xlbs.onrender.com)
